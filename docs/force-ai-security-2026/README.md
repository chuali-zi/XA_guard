## 核心判断

这组材料的主线不是单纯的“提示注入防御”，而是企业进入智能体时代后，安全边界从模型输入输出扩展到 **身份、权限、工具、数据路径、基础设施、运行时编排、审计责任和韧性恢复** 的全链路治理。

用户的现场印象“投毒不是唯一重点，更重要的是多 Agent 管理、信息数据处理和治理安全”是准确的。照片中的多个主题都在强调：

- 智能体已经成为“第三类身份”：既不像人类账号，也不同于传统静态服务账号。
- 风险从单点漏洞下沉到数据路径全链：恶意内容进入上下文、被 Agent 当作可信语境处理，再触发工具执行，最终造成数据外泄或资金转移。
- Computer Use Agent 会绕过浏览器时代的传统边界：同源策略、站点隔离、CSRF、Cookie、人工确认等防线在“代人看屏幕、代人操作”的模式下会失效或弱化。
- 企业最大问题不只是模型安全，而是 Agent Gateway、身份治理、MCP / Skills 授权、日志审计、数据隔离、供应链与可恢复能力。
- 可信智能体的目标函数可以概括为：在信任与安全约束下最大化意图完成，而不是单纯提高模型能力。

## 一句话内化

如果把这组 PPT 转成 XA-Guard 的产品语言，重点应从“拦截提示注入”升级为：

> 通过 Agent Gateway 对每个智能体建立可认证身份、最小权限、任务级授权、数据路径隔离、工具调用审计、异常行为阻断和动作级撤销，让企业能够安全地使用多 Agent 编排。

## 对赛题的直接启发

本仓库当前的六关卡思路与会议材料高度同频，但要在答辩和交付物中更明确表达“Agent 治理”：

- Gate1 不只是提示注入检测，而是外部内容进入可信上下文前的第一道边界。
- Gate2 / Gate3 不只是工具审批和策略规则，而是 Agent 权限、JIT/JEA/JLA 和 Agent Gateway 的落地点。
- Gate4 不只是内容安全，还应覆盖 PII、数据外泄、上下文污染、行业话题边界和模型输出可信性。
- Gate5 不只是沙盒，而是把工具、插件、MCP、RAG、文件 IO、网络 IO 的 blast radius 控住。
- Gate6 不只是日志，而是“法律责任、商业风险量化、可追溯、可撤销、可复盘”的证据底座。
