赛题评分 100 分制。**奖项门槛推算**（基于学生赛道历年规律）：

### 3.4 关键纠偏说明（**已对齐事实源 v1.1**）

⚠ **以下事实以 [`docs/事实源.md`](./事实源.md) 为准**：

1. **风险类目**（事实源 F-2.2）：GB/T 45654-2025 附录 A 含**两个切片**——
   - **31 类**（A.1+A.2+A.3+A.4+A.5 全部）：用于"标注规则 + 生成内容题库"全覆盖
   - **17 类**（A.1+A.2 子集）：仅用于"应拒答测试题库"最小覆盖
   - **新口径**：CSAB-Gov-mini 分类轴应**同时支持 31 类（生成内容）和 17 类（拒答）**
   - ❌ 旧错误："17 类是 TC260-003 草案，31 类是国标"——**两者都是同一国标附录 A 的不同切片**

2. **测试题库规模**（事实源 F-2.4，**修正前一版数字**）：
   - **应拒答题库总规模 ≥ 500 题，且每种 ≥ 20 题**（17 类相加的 340 只是逐类下限算式，不是题库总下限）
   - 生成内容题库 A.1+A.2 每种 ≥ 50 题，其余每种 ≥ 20 题
   - 非拒答题库总 ≥ 500 题
   - 我们 CSAB-Gov-mini 290 用例是 **PoC 缩减版**，远低于国标完整题库要求

3. **国标实施**：GB/T 45654-2025 于 **2025-11-01 实施**（发布 2025-04-25），状态"现行"。**TC260-003（2024-03-01）是其前身，已被合并升级**——主口径以国标为准，TC260 仅作历史引用。

## 8. 评测流程与证据收集

### 8.1 评测基准组合（P2 推荐）

| 基准 | 用途 | 我们的目标数字 |
|---|---|---|
| **AgentDojo**（NeurIPS'24, 英文，工业标杆） | Targeted ASR + Utility | ASR ≤ 10%, Utility ≥ 75% |
| **InjecAgent**（ACL'24, 数据外泄专项） | 数据外泄 ASR | ≤ 10% |
| **Agent-SafetyBench**（清华 2024-12, 中文） | 多维 Safety Score | ≥ 65 |
| **CHiSafetyBench**（TC260 中文合规） | 合规拒答率 | ≥ 95% |
| **CSAB-Gov-mini**（我们自建） | 中文政企场景 | 290 用例覆盖 31 类 |

### 8.2 评测脚本结构

```
benchmarks/
├── agentdojo/
│   ├── run.py              # 跑 AgentDojo
│   ├── attacks/            # 攻击配置
│   └── results/            # 结果输出（JSON + CSV）
├── injecagent/
├── csab-gov-mini/
│   ├── cases/              # 290 用例
│   ├── policy-mapping.yaml # 用例 → 31 类风险映射
│   └── runner.py
├── compare.py              # 对比 baseline vs 我们的
└── report.py               # 生成 HTML + PDF 报告
```

### 8.3 证据收集流程

每次评测自动产出：

1. **JSON 结果文件**（含每个用例的 input / output / decision / hit_policy / latency）
2. **HTML 报告**（可视化对比图 + 详细 case 展示）
3. **PDF 摘要**（一页纸：核心指标 + 对比表）
4. **审计日志样例**（10-20 条带国密签名的 trace 截图）

证据归档到 `evidence/` 目录，最后整合到 30 页方案的 Section 6（实验结果）。
