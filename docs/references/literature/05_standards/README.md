## 这个方向研究什么

**通俗解释**：政企单位（政府、国企、央企、医院、银行……）部署任何信息系统，**必须符合国家法律、行政法规、国家标准**——不达标就别想上线。

大模型智能体作为新型信息系统，同样要面对这套合规体系。我们项目想被政企客户接受，**合规对标**是硬门槛。

## 与我们项目 6 关卡的对应关系

### 关卡 3 · 规则编译器（核心创新）

这个关卡的**输入源**就是本目录的合规标准。具体规则映射：

| 合规来源 | 编译为机器规则 |
|---|---|
| **GB/T 45654-2025 附录 A.1+A.2（17 类）应拒答**（主口径国标；TC260-003 为前身已合并） | 内容审查 OPA 规则 |
| **GB/T 45654-2025 附录 A 全集（31 类）生成内容审查** | 内容审查 OPA 规则 |
| 等保 2.0 三级"重要操作授权" | HITL 三色审批规则 |
| 等保 2.0 三级"安全审计 6 个月留存" | 关卡 6 留存策略 |
| 等保 2.0 三级"访问控制" | Policy DSL 访问规则 |
| GB/T 39786 密码应用要求 | 关卡 6 国密签名 + 哈希链 |
| AI 服务管理办法第 X 条 内容标识 | 输出层水印 + 标识 |

### 关卡 6 · 黑匣子审计

- 等保 2.0 → 审计记录留存 6 个月
- GB/T 39786 → SM3 哈希链 + SM2 签名
- AI 服务管理办法 → 内容标识 + 应急响应记录

### 评测基准 · 考场

- **GB/T 45654-2025 17 类应拒答**（A.1+A.2 切片）→ 应拒答测试用例（每种 ≥ 20 题，总规模 ≥ 500 题）
- **GB/T 45654-2025 31 类生成内容审查**（A 全集）→ 生成内容测试用例（A.1+A.2 每种 ≥ 50 题，其余 ≥ 20 题）
- OWASP LLM Top 10 → 每类设计 5-10 个测试用例
- NIST AI 600-1 12 类风险 → 国际对标测试维度
