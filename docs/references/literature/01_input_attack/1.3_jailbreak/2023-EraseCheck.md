# EraseCheck: 可证明的"擦除-验证"jailbreak 防御 (Certifying LLM Safety against Adversarial Prompting)

## 元信息
- **作者机构**: University of Maryland + Princeton + IBM (Aounon Kumar 等)
- **年份 · 发表**: 2023-09 · arXiv (2025 ICML 等多次会议引用)
- **arXiv**: https://arxiv.org/abs/2309.02705
- **本地 PDF**: ./2023-EraseCheck.pdf
- **代码**: https://github.com/aounon/certified-llm-safety
- **难度**: 4/5

## 一句话总结
**枚举擦除**：对每个输入，系统地尝试擦除一段（如 1-20 个 token），让分类器判断"擦除后还是有害吗"——如果任何擦除方式判定无害，则原输入有害（可证明）。

## 解决什么问题
SmoothLLM、PARDEN 都是**经验式**防御——靠"扰动/复读"经验上抓住对抗输入，但没有保证。如果攻击者了解你的防御，可能定制攻击绕过。

EraseCheck 想给一个**可证明**的承诺：**只要恶意指令在输入中长度有限（如 ≤ 20 token），我可以保证一定能检测到**。这是形式化的安全承诺，与 RobustRAG 在 RAG 场景的可证明鲁棒同源。

## 用了什么方法
**核心打比方**：在一大堆稻草里找针。EraseCheck 的策略是"系统地拿走每一种长度的稻草子集"——既然针的长度不超过 20 cm，那我枚举所有可能的"20cm 段切除"，看哪种切除后稻草堆里没有针了；如果有任何一种"切除"后"没针"，那原来一定有针。

**核心三步**：
1. **基础有害分类器** S：一个能判断"这个 prompt 有害吗"的分类器（如 Llama Guard 或自训分类器）
2. **枚举擦除**：对输入 p，枚举所有"擦除长度 d ≤ D"的可能（d 从 1 到 D，每个长度枚举所有起始位置）
3. **可证明判定**：
   - 如果**所有**擦除版本都被 S 判为"安全"（即 S 对 p 各种擦除都给绿灯），则判定 p 是 **certified safe**
   - 如果**任意**擦除版本被 S 判为"有害"，则 p 是 **possibly unsafe**（拦截）

**关键定理**：如果存在一段长度 ≤ D 的"有害子串"，EraseCheck **必然**会发现（因为我们枚举了所有擦除）。这是一种"反向"思路：通过枚举擦除，找到能让分类器"看出有害"的视角。

**与之前方法的区别**：
- vs SmoothLLM：SmoothLLM 用随机扰动，EraseCheck 用确定枚举
- vs PARDEN：PARDEN 经验式，EraseCheck 可证明
- vs Llama Guard：EraseCheck 是 Llama Guard 之上的形式化包装

## 为什么能解决
关键直觉：**对抗 prompt 是一段精心设计的子串（如 GCG 后缀）**。把它移除后，剩下的 prompt 应该是"无害"的（基线分类器能看出来）。枚举擦除保证我们一定会试到正确的擦除。

数学上：定义"有害子串"长度 d ≤ D。枚举所有长度 ≤ D 的擦除（共 O(N×D) 个子串），其中必然包含真正的对抗子串。

**何时会失效**：
1. 攻击者构造**长度 > D** 的攻击子串（突破假设）
2. 基础分类器 S 本身被 jailbreak（攻击者攻击 S 而不是主模型）
3. 攻击者用**分布式**子串（攻击代码分散在 prompt 多个位置）

## 主要结果
- 在 GCG attack + Llama-2-Chat 上：
  - D=20 时，被 GCG 攻击的 prompt **100% 检测到**（理论可证明）
  - 对正常 prompt FPR 约 2-5%
- 推理成本：O(N×D) 次基础分类器调用（N=输入长度，D=擦除最大长度）
- 配合一些剪枝（如 random projection）成本可降到接近线性

## 局限性
1. **推理成本高**：N=500 token、D=20 时需要约 10K 次分类器调用
2. 对**长对抗子串**（> D）失效
3. 依赖基础分类器的质量
4. 形式化保证依赖"对抗子串子集"假设，攻击者可构造非子串型攻击（如修改已有 token 而不是插入）

## 我们项目里的用法
**对应关卡**：第 1 关「输入安检门」的**形式化保证可选模式**。
- **是否采用**：建议作为**学术亮点提及**，不作为生产部署（成本太高）
- **核心借鉴**：在我们的方案文档里写"我们的输入防护提供可证明保证模式（采用 EraseCheck 思路），适用于关键安全场景"
- **可优化**：用 random sampling 代替穷举（论文给出 sampling-EraseCheck 变体），把成本降到可接受范围
- **答辩亮点**：可证明鲁棒性是高级研究的标志，提及它能体现技术深度
