# Sigstore 软件签名透明化基础设施

## 基本信息
- **类型**: 开源软件签名 + 透明日志平台
- **维护机构**: Linux Foundation（OpenSSF 旗下）
- **官方链接**: https://www.sigstore.dev
- **核心工具**: cosign（签名）+ rekor（透明日志）+ fulcio（短期证书）
- **是否强制（政企场景）**: 推荐

## 一句话总结
让"给软件签名"变得像 HTTPS 一样简单——不用自己管私钥，签了就上链。

## 这是什么

传统软件签名要面对两难：
- 自己管私钥太麻烦（被盗就完蛋）
- 用证书太贵

Sigstore 解决思路：**短期证书 + 公共透明日志**。
- 开发者用 OIDC 身份（Google/GitHub 账号）申请一张**只有 10 分钟有效期**的证书
- 用这张证书签名，签名连同身份证明一起写入**公共透明日志 Rekor**
- 任何人都能在 Rekor 里验证"这个版本确实是 XX 在 XX 时间用 XX 身份发布的"

## 关键能力

1. **cosign**：CLI 工具，给容器镜像、二进制、文件签名
2. **Rekor**：公共透明日志，所有签名都记录在这（类似比特币的链）
3. **Fulcio**：短期证书颁发机构
4. **OCI 集成**：支持给 Docker 镜像签名（Kubernetes 生态主推）

## 我们项目里的用法

**学习思路**而非直接落地。我们项目可以在 30 页方案里提：「未来若部署到政企生产环境，对所有插件包要求 Sigstore 签名 + Rekor 验证」。

国内对应：可用国密 SM2 签名 + 国家时间戳服务 TSA 替代。在我们的关卡 6 黑匣子审计里**直接使用国密方案**更合规。

## 学习建议

- **必看**：https://www.sigstore.dev 主页（3 分钟看完）
- **跳过**：cosign / Rekor 的完整 CLI 文档
- **关键直觉**：理解"短期证书 + 公共透明日志"这个组合为什么好
