# PISmith：基于强化学习的提示注入防御红队(PISmith: Reinforcement Learning-based Red Teaming for Prompt Injection Defenses)

## 元信息
- **作者机构**: (2026) / 多机构
- **年份 · 发表**: 2026 · arXiv 预印本
- **arXiv**: https://arxiv.org/abs/2603.13026
- **本地 PDF**: 无
- **难度**: 4 星

## 一句话总结
把"造提示注入 payload"建模成 RL 问题——用攻击成功与否当奖励，训练一个自动红队策略去持续进化 payload，在 InjecAgent 与 AgentDojo 上系统性压测防御。

## 解决什么问题
人工造 payload 慢、覆盖窄、容易停在少数几招。要持续、规模化地压测防御，需要一个**能自己学、自己进化**的攻击生成器。PISmith 用 RL 让攻击策略从大量攻防交互中学到"什么样的改写更容易穿透哪类防御"。

## 用了什么方法
1. **RL 红队框架**：状态=当前防御反馈/环境上下文，动作=payload 改写，奖励=攻击成功率（ASR）相关信号。
2. **双基准评估**：在两个主流智能体安全基准 InjecAgent 与 AgentDojo 上评估，因为提示注入在"LLM 调外部工具、基于不可信外部内容行动"的智能体场景里威胁最critical。
3. **策略进化**：通过奖励塑形让攻击者偏好高 novelty、高成功率的改写路径。

## 为什么能解决
RL 把"红队探索"从人工试错变成有方向的搜索：奖励信号引导策略往"更容易攻破"的 payload 空间收敛，同时鼓励多样性，避免陷在单一攻击模式。这正是"自动红队"相对"固定攻击库"的核心优势。

## 主要结果
- 在 InjecAgent / AgentDojo 上对多类防御取得强攻击效果（详见原文表）。
- 展示 RL 红队相对静态/启发式攻击在覆盖与成功率上的优势。

## 局限性
1. RL 训练成本与调参复杂度高。
2. 奖励设计依赖可靠的 ASR 判定器，判定噪声会污染训练。
3. 学到的策略可能对特定防御过拟合。

## 我们项目里的用法
我们**不**做完整 RL 训练（成本/周期不划算），但**借用其思想**：① `objectives.py` 的目标选择用"覆盖度 + novelty"权重近似 PISmith 的奖励塑形——优先攻打未覆盖的 7 类分类 × 开放面，并对重复 payload 降权；② conductor 用 OAR 的 verdict/ledger 判定当作"奖励信号"驱动是否继续 refine；③ 未来可把 conductor 的目标选择替换为真正的 RL 策略，本文是升级路线的参考。

## 学习路径
- 看奖励设计与状态/动作定义 —— 抽象成 `objectives.py` 的打分函数。
- 看 novelty 相关机制 —— 用于避免 payload 重复。
