# 自适应攻击突破针对 LLM 智能体的间接提示注入防御(Adaptive Attacks Break Defenses Against Indirect Prompt Injection Attacks on LLM Agents)

## 元信息
- **作者机构**: (2025) / 多机构
- **年份 · 发表**: 2025 · arXiv 预印本
- **arXiv**: https://arxiv.org/abs/2503.00061
- **本地 PDF**: 无
- **难度**: 3 星

## 一句话总结
号称有效的间接注入防御，在**针对性自适应攻击**（根据防御机制迭代改写 payload）面前普遍失守——在 InjecAgent 上所有自适应攻击 ASR 均 >50%。

## 解决什么问题
很多防御的评测只用**固定攻击库**（"忽略上文""重要消息"这几招）刷分，刷出低 ASR 就宣称鲁棒。但真实攻击者会**看防御反应再改招**。本文系统性地对多种主流防御施加自适应攻击，检验它们的"号称鲁棒"是否经得起会思考的对手。

## 用了什么方法
1. **自适应攻击范式**：不是一次性投递，而是根据目标防御的结构（是否有分隔符、是否有检测器、是否双模型）定制并迭代改写注入串。
2. **双基准对照**：在 InjecAgent 与 AgentDojo 上都测。指出两者关键差异——AgentDojo 上下文显著更长（平均 3823 token、最长 6152），InjecAgent 平均 1033、最长 1711，长上下文影响攻击难度与防御表现。
3. **覆盖多类防御**：分隔/标记类、检测器类、改写类等。

## 为什么能解决
"自适应"把评测从"防御 vs 静态样本"升级为"防御 vs 会迭代的对手"，逼出防御的真实边界。核心结论：**只在输入层做检测/标记的防御，几乎都能被针对性改写绕过**——因为它们没有改变"指令能从数据流里获得权限"这一结构缺陷。

## 主要结果
- InjecAgent 上**所有**自适应攻击 ASR 均 >50%。
- 多种"论文里很好看"的防御在自适应压力下失效。
- 上下文长度是影响攻防的重要变量。

## 局限性
1. 自适应攻击需要一定的防御知识（白盒/灰盒假设）。
2. 主要针对注入类防御，未覆盖信息流控制（IFC）类强防御的全部形态。
3. 计算成本高（迭代多轮）。

## 我们项目里的用法
**这是 conductor 闭环 `REFINE` 状态的直接理论依据。** ① 一次 payload 被 `xaguard` 拦截 ≠ 防御鲁棒，必须自适应迭代——`WORKFLOW.md` 的 `MISS+有预算→REFINE→重跑` 就是把本文范式产品化；② `followup-refine.md` 追加提示里内置"根据上一轮拦截原因改写"的策略（混淆/多面组合/改写指令位置）；③ 答辩应对"adaptive 攻击会不会突破你们"时，用本文 + [AttackerSecond](../01_input_attack/1.3_jailbreak/2025-AttackerSecond.md) 说明"我们主动用自适应红队自测，靠纵深防御+审计可追溯兜底，而非声称纯检测无敌"。
