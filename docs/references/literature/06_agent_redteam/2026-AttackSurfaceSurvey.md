# 分层攻击面综述：LLM 智能体安全威胁与防御系统性综述(A Systematic Survey of Security Threats and Defenses in LLM-Based AI Agents: A Layered Attack Surface Framework)

- **年份 · 发表**: 2026 · arXiv 预印本

## 一句话总结
把智能体安全威胁**按架构分层**而非按攻击类型堆砌，指出千变万化的攻击共享一个根因——**principal trust inversion（主体信任反转）**，并给出跨层防御配方。

## 解决什么问题
此前的智能体安全分类学都**按攻击类型排序**（提示注入、越狱、投毒……），但这只回答"是什么攻击"，不回答"打的是哪一层架构"。而智能体与无状态 LLM 的本质区别恰恰是**架构**：它跨多会话规划、保留记忆、调用外部工具、与同伴 agent 协作。攻击类型分类学对"防御该插在哪一层"毫无指导。本综述用 PRISMA 方法筛出 116 篇一级安全论文，改用**架构层**（感知/记忆/工具/多agent/时序）来组织威胁。

## 用了什么方法
1. **分层攻击面框架**：把威胁归到具体架构层，每层有独立的安全属性。
2. **记忆四层模型**：in-context memory（活跃 token 窗，会话内可被 context stuffing / distraction 操纵）、工作记忆、长期记忆、经验检索池——各层安全性质不同。点名 2025–2026 新攻击 **MemoryGraft**（投毒经验检索池，持久污染未来工具选择）。
3. **根因归约**：几乎所有攻击都归结为单一结构性缺陷——**principal trust inversion**：智能体错误地把不可信数据源（网页/文件/API 返回/项目元数据）里夹带的指令当作可信主体（主人）的指令来执行。攻击面的多样性（口子在哪）不如根因的统一性重要。
4. **跨层防御分类学**：对 7 类典型攻击给出处方式防御配方，并用依赖 DAG 区分"工程可解的缺口"与"根本性开放问题"。

## 为什么能解决
按层分类让"防御插在哪"变得可判定：知道攻击打的是记忆层，就知道该上记忆隔离/来源标记而不是输入过滤。根因归约让防御设计有了统一靶心——只要能维持"指令只能来自可信主体"这一不变量，一整类攻击就被封死。

## 主要结果
- 提炼出被广泛引用的 **7 类攻击**：①提示注入 ②记忆攻击 ③工具与协议攻击 ④多智能体攻击 ⑤多模态攻击 ⑥工具链/供应链攻击 ⑦时序攻击。
- 论证记忆层是最被低估的持久化攻击面（MemoryGraft 类持久污染）。
- 给出"哪些是工程可解 vs 根本开放"的依赖图。

## 局限性
1. 综述不含新实验数据，结论依赖被综述论文的原始设定。
2. 7 类划分在多模态/时序上覆盖仍浅。
3. principal trust inversion 是有力的直觉框架，但缺少可形式化验证的统一防御证明。
