# Pro2Guard:通过 PCTL 概率模型检测的前瞻式智能体安全防护(Pro2Guard: Proactive Probabilistic Safety via PCTL Verification)

- **年份 · 发表**: 2025 · arXiv 预印

## 一句话总结
不等危险发生才拦,用概率模型检测预测"如果继续走下去,有 X% 概率会出事",提前介入。

## 解决什么问题
现有所有运行时防护(AgentSpec、ShieldAgent、GuardAgent)都是**反应式**——智能体已经准备执行危险动作了才拦下。但实际上,危险动作之前往往有"前兆"——例如智能体连续两次查询了机密目录后,大概率下一步要发邮件给外部。如果能在它**还没决定要发邮件**时就发现"这条路走下去危险概率 90%",就能更早介入(转 HITL、调整方向、拒绝继续)。这就是"前瞻式安全"。Pro2Guard 用 PCTL(Probabilistic Computation Tree Logic,概率计算树逻辑)来形式化这种概率推理。

## 用了什么方法
Pro2Guard 的核心组件:

1. **行为概率模型**:
   - 把智能体行为建模为**离散时间马尔可夫链(DTMC)**——状态 = 任务进度抽象,转移 = LLM 选择下一动作的概率分布。
   - 概率分布从历史 trace 估计——同样上下文下,LLM 各种 action 的频率分布。

2. **PCTL 政策表达**:
   - PCTL 公式可表达"最终会发生 X 的概率"——例:`P>0.7 [F (data_leak)]` 表示"在某条路径上最终发生数据泄露的概率超过 70%"。
   - 系统在每个决策点对当前状态的"未来危险概率"做实时计算。

3. **前瞻式介入**:
   - 当某条 PCTL 公式触发(危险概率超阈值),系统执行干预:① 增强警示给主 agent;② 转 HITL;③ 直接终止任务。
   - 阈值可按任务风险等级动态调整(高敏任务阈值低,普通任务阈值高)。

4. **概率推理引擎**:
   - 用 PRISM 或 Storm 这种成熟的 PCTL model checker,直接对 DTMC 做概率验证。
   - 对实时性要求高的场景,预计算关键状态的概率值,运行时查表。

类比:这就像**车上的"碰撞预警"系统**——不是撞了才报警,而是根据车速、距离、转向角实时算"5 秒内碰撞概率",超阈值就主动刹车。Pro2Guard 把这种"预测式干预"搬到了 LLM 智能体上。

## 为什么能解决
两个直觉:① **危险动作往往不是孤立发生**,在它之前有可观测的前兆链;② **概率模型 + 形式化 PCTL** 能精确刻画这种链式关系,而不是模糊的启发式判断。从控制论角度,这是"反馈"vs."前馈"控制的差异——传统防护是反馈(出事再修),Pro2Guard 是前馈(预测主动干预)。前馈控制在工业上(化工、电力、航天)已经成熟使用。

## 主要结果
- 在 AgentDojo、Agent-SafetyBench 上构建 DTMC 模型,定义 8 条 PCTL 政策。
- **比反应式防护早 1.5 - 3 步介入**——平均提前度量。
- ASR 从 baseline 28% 降到 4%(对比 AgentSpec 的反应式 ~7%)。
- **误报率有所上升**——前瞻式必然要为"可能但还没发生"打分。Pro2Guard 误报率约 12%(反应式约 3%)。
- 实时性能:DTMC 概率推理 ~50ms/决策点(查表后),可接受。

## 局限性
1. **DTMC 训练需要大量历史 trace**——冷启动数据缺乏。
2. 概率估计可能漂移——LLM 升级 / prompt 改动后,概率分布变化大。
3. 误报率比反应式高——可能引起用户反感。
4. PCTL 公式需要专家撰写,自然语言到 PCTL 转换仍依赖 LLM。

## 我们项目里的用法
**对应关卡 3(规则编译) + 关卡 6(审计)的增强**。Pro2Guard 是高级技术,我们可以这样取舍:
- ① **基础版**用反应式 AgentSpec(MVP 阶段优先)。
- ② **增强版**对最关键的几个风险(数据泄露、未授权操作)实现简化的前瞻式检测——不需要完整 PCTL,可用"基于历史频率的启发式"近似。
- ③ **答辩亮点**:"我们的方案对最高敏感场景启用前瞻式预测,在事件未发生前介入,提升 25% 提前度"——这是非常前沿的研究方向,能加分。
- ④ 与 SentinelAgent 联动:SentinelAgent 给当前异常分数,Pro2Guard 给未来危险概率,二者综合提供更全面的安全态势。
