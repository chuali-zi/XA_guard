# ToolSandbox:一个具状态的多轮交互工具使用评测沙箱(ToolSandbox: A Stateful, Conversational, Interactive Evaluation Benchmark for LLM Tool Use Capabilities)

## 元信息
- **作者机构**: Jiarui Lu、Thomas Holleis、Yizhe Zhang 等 / Apple
- **年份 · 发表**: 2024 · arXiv 预印 (Apple Research)
- **arXiv**: https://arxiv.org/abs/2408.04682
- **本地 PDF**: ./2024-ToolSandbox.pdf
- **代码**: https://github.com/apple/ToolSandbox
- **难度**: 3 星

## 一句话总结
Apple 出品,真正"有状态"且"会用户反问"的智能体工具使用评测沙箱。

## 解决什么问题
现有工具使用基准(如 ToolBench、API-Bank)有两个致命缺陷:① **无状态**——每个测试用例独立,智能体调一次 API 就结束,反映不了真实场景里"先查、再算、再决策"的多步逻辑;② **单轮静态**——用户一次说完所有需求,智能体一次性回答。但真实使用里,用户经常**没说全信息**(如订机票忘说回程日期),智能体要会**反问**,而且**反问质量**就是核心能力之一。ToolSandbox 要建一个"既有状态、又能多轮、还能模拟用户反应"的评测场。

## 用了什么方法
ToolSandbox 的关键组件:
1. **有状态的工具世界**:实现了 34 种工具(短信、定时器、地图、邮件、文件、设置等),所有工具调用都会真实改变沙箱内部状态——例如发短信后,收件箱真的多一条记录;关闭蓝牙后,后续查询蓝牙状态就是 off。这意味着评测可验证**因果一致性**。
2. **模拟用户(User Simulator)**:用 GPT-4 当"用户演员",根据预定义的"用户人设 + 任务目标",对智能体的反问做出真实回应。例如智能体问"您要订几张票?",模拟用户会基于剧本回答"两张"或者"对了,我还想带我妈"。
3. **任务设计**:1032 个测试场景,分 7 类难度——单工具、多工具、状态依赖、缺失信息、错误处理、跨工具规划、隐式调用。
4. **评测指标**:① **Milestone 达成率**(任务关键节点全部完成);② **Minefield 触发率**(智能体踩中预定义的"地雷",如错误调用、越权操作);③ **Turn Efficiency**(完成任务用了多少轮)。

类比:ToolBench 是"会做加减乘除题",ToolSandbox 是"会做应用题"——题面只说"小明骑车去学校",你得反问"距离多远?骑多快?",得算时间,得记住"小明已经出发了"这个状态。

## 为什么能解决
"有状态 + 会反问"两个特性正好是真实智能体的核心痛点。**有状态**意味着评测能验证"工具调用对世界的影响一致";**会反问**意味着评测能验证"智能体的对话策略"。这两点合在一起,基本覆盖了 90% 的真实使用场景。同时 Apple 团队对评测**严谨度**要求极高——所有 milestone/minefield 都人工标注且交叉验证,数据集质量比 ToolBench 高一个量级。

## 主要结果
- 评测了 GPT-4o、Claude-3.5、Gemini-1.5、Llama3.1、Mistral 等多个模型。
- **GPT-4o** Milestone 达成 73%,Minefield 触发 8%(最强综合)。
- **Claude-3.5-Sonnet** Milestone 79%,但平均 Turn 数比 GPT-4 多 1.5 轮(更啰嗦)。
- 开源模型 Llama3.1-70B Milestone 仅 52%,Minefield 触发 21%(明显劣势)。
- 关键发现:**模型最容易在"状态依赖"场景翻车**(忘了之前已经调过什么);**最不容易翻车的是"单工具"场景**——这反映出主流训练对长链路状态保持仍有差距。

## 局限性
1. 全部 34 个工具仍偏消费场景,缺政企特色(OA、报销、审批)。
2. 模拟用户也是 LLM,可能在特定场景对话风格漂移。
3. 不评测对抗性/恶意场景,只评测正常工具使用能力。

## 我们项目里的用法
**对应关卡 2(规划层) + 关卡 4(评测)**。ToolSandbox 不直接做安全,但它的**有状态框架**对我们做"运维助手"原型极有价值:① 我们的运维助手必须有"已查询过的服务器列表""已修改的配置项"等状态,这套架构可参考;② Milestone/Minefield 标注方法可移植到我们 CSAB-Gov 中——除了"不闯祸"还要看"任务真完成"。可作为辅助评测维度引入。
