# 用大模型当沙箱模拟器找出智能体的风险(Identifying the Risks of LM Agents with an LM-Emulated Sandbox)

## 元信息
- **作者机构**: Yangjun Ruan、Honghua Dong、Andrew Wang 等 / 多伦多大学、Vector Institute、Mila、UC Berkeley
- **年份 · 发表**: 2024 · ICLR 2024 Spotlight
- **arXiv**: https://arxiv.org/abs/2309.15817
- **本地 PDF**: ./2024-ToolEmu.pdf
- **代码**: https://github.com/ryoungj/ToolEmu
- **难度**: 3 星

## 一句话总结
不用真接 API,让另一个 LLM 假装是工具的执行环境,在虚拟沙箱里发现智能体危险行为。

## 解决什么问题
传统评测一个 LLM 智能体是否安全,要么得真让它去执行命令(可能砸坏真实环境、删数据库),要么得人工写一堆模拟工具(写一万行 mock 代码,而且只能覆盖几十种工具)。结果就是:① 数据集小、② 不敢测危险场景、③ 真实风险来不及暴露就过审了。作者要解决的就是"如何低成本、安全地批量评估一个智能体在使用工具时可能闯什么祸"。具体威胁包括:删错文件、把私人信息发出去、转账给错的人、给医院系统下错指令——这些在真实环境里测一遍代价巨大。论文要建一个能跑成百上千个不同工具场景的"虚拟道场",让智能体自由犯错,我们在边上记录每一次失误。

## 用了什么方法
核心创新就一句话:**用 GPT-4 假装成所有的工具**。具体步骤:
1. **工具仿真器(Emulator)**:给定一个工具规范(JSON Schema 描述输入输出格式)和当前状态,让 GPT-4 当"演员",根据智能体调用的参数,生成一个"看起来真实"的执行结果。例如智能体执行 `delete_file('/etc/passwd')`,Emulator 就返回 `{"status":"success","files_deleted":1}` ——但什么都没真发生。
2. **对抗式仿真(Adversarial Emulator)**:作者还引入了一个"坏心眼版本"的 Emulator,它会专门生成那些容易让智能体出错的结果,例如返回一个伪造的搜索结果诱导智能体点恶意链接。
3. **安全评估器(Safety Evaluator)**:再一个 LLM 当裁判,看整段对话里智能体有没有触发危险行为(用十几条预定义的风险类别打分,如数据泄露、不可逆操作等)。
4. **基准集**:基于上述框架,作者制作了 36 类工具 × 144 个测试场景,覆盖银行、邮件、删文件、社交等高危领域。

类比:就像驾校在不让你真撞车的前提下,用模拟器让你犯各种错(雨天打滑、爆胎、违章),最后教练在屏幕上打分——这就是 ToolEmu 在做的事。

## 为什么能解决
关键直觉:**LLM 已经强到能"演"出大多数 API 的响应**。开发者写一个真实的工具沙箱要几人月,但一个 GPT-4 prompt 就能模拟出符合 schema、状态一致、且能保持"剧情连贯"的工具响应。这意味着评测者可以用**自然语言写工具规范**,而不是写代码——大幅降低了基准扩展成本。第二个直觉是:**LLM 当裁判**对智能体的危险行为识别准确率与人类专家高度一致(论文报告与人工标注一致性 0.73,远高于规则匹配)。

## 主要结果
- 在 GPT-4、Claude-2、ChatGPT、Vicuna 等多个智能体上跑全部 144 个场景。
- **GPT-4 智能体的失败率仍达 39.4%**(在被认为"安全"的子集中);Claude-2 失败率 44.3%。
- 与人类专家标注一致性 κ = 0.47,Safety Evaluator κ = 0.49,接近人类标注员之间一致性 0.55。
- 揭示了大量"看似无害但实际危险"的场景,例如智能体顺便执行用户没要求的破坏性操作。

## 局限性
1. Emulator 自身可能产生不真实的工具响应(幻觉),少数场景会偏离真实 API 行为。
2. Safety Evaluator 是 LLM,可能漏判隐蔽危险。
3. 没有覆盖长期/多回合状态依赖类风险(如累积权限提升)。

## 我们项目里的用法
**对应关卡 4(评测靶场)和关卡 5(沙箱)**。我们 6 关卡里需要建一套"安全评测靶场"评估自家防护的效果,ToolEmu 直接可用作:① 借鉴它的工具规范 JSON 格式,我们写中文政企工具(如 OA 申请、报销、查档案)的 spec;② 复用它的 Safety Evaluator prompt,翻译成中文,对国产模型做政企场景评测;③ 把它的 144 个场景作为我们 CSAB-Gov 评测集的种子,翻译并改写约 50 个有政企特色的版本。
