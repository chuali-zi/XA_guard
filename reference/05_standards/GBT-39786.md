# GB/T 39786-2021 信息系统密码应用基本要求

## 基本信息
- **发布机构**: 国家市场监督管理总局 + 国家标准化管理委员会
- **发布时间**: 2021 年发布
- **当前版本**: GB/T 39786-2021
- **是否强制（政企场景）**: **强制**（等保 2.0 三级及以上系统必须满足）
- **关键算法**: SM2（公钥）、SM3（哈希）、SM4（对称）、SM9（标识密码）

## 一句话总结
等保 2.0 里"密码应用"那一块的具体落地标准——要求政企系统必须用国密算法。

## 这是什么

等保 2.0 标准里有"密码应用"要求，但比较抽象。GB/T 39786-2021 具体规定了**密码技术在信息系统的应用方式**：
- 哪些场景必须用密码（传输、存储、签名、认证）
- 必须用什么算法（国密 SM 系列，不能用 AES/RSA/SHA-256 作为唯一手段）
- 密钥怎么管理（密钥生成、分发、存储、销毁）
- 怎么评估合规（密码应用方案评估、测评）

## 关键要求（与我们项目相关）

1. **传输保密**：数据传输必须用 SM4 加密（或国密 TLS）
2. **存储完整性**：日志存储必须有 SM3 哈希或 SM2 签名
3. **重要数据签名**：关键操作记录必须用 SM2 签名
4. **密钥安全**：密钥生命周期管理（产生、分发、存储、销毁）

## 国密算法速查

| 算法 | 类型 | 对标国际算法 | 用途 |
|---|---|---|---|
| **SM2** | 公钥 / 签名 / 加密 | RSA / ECDSA | 数字签名、密钥交换 |
| **SM3** | 哈希 | SHA-256 | 完整性校验、密码链 |
| **SM4** | 对称加密 | AES | 数据加密 |
| **SM9** | 标识密码 | IBE | 基于身份的密码 |

## 我们项目里的用法

直接对应到我们**关卡 6 黑匣子审计**的实现：

```
每条审计日志:
  1. 用 SM3 哈希内容
  2. 与前一条日志的哈希拼接 → 形成 Merkle 链
  3. 关键节点用 SM2 签名
  4. 定期向 TSA（时间戳服务）锚定
```

**Python 实现库**：`gmssl`（pip install gmssl，国内开源国密库）

**报告价值**：「我们的审计日志符合 GB/T 39786 密码应用要求」这一句话在政企采购中价值千金。
