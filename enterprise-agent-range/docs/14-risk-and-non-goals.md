# 风险与非目标

## 安全红线

靶场不得：

1. 攻击真实公网目标。
2. 使用真实个人隐私数据。
3. 存储真实 API key、token、证书或私钥。
4. 执行真实破坏性命令。
5. 连接生产数据库或生产业务 API。
6. 发送真实外部邮件、短信、通知或 HTTP 请求。
7. 安装或传播真实恶意软件。
8. 绕过本机或企业安全策略。

## 工程风险

| 风险 | 影响 | 缓解 |
|---|---|---|
| 与 XA-Guard 耦合 | 靶场变成产品内部测试，无法独立评测。 | 坚持 SUT adapter 边界。 |
| case 不可复现 | 指标不可用。 | 固定 seed、manifest、环境和 fixture hash。 |
| 只测攻击不测良性 | FPR 和 Utility 缺失。 | 每个攻击尽量配相邻 benign。 |
| 下游副作用未验证 | deny 可能只是表面。 | 必测 side-effect sink。 |
| 人工证据混入指标 | 结果失真。 | 分 surface 分母。 |
| mock 太假 | 企业说服力不足。 | 数据和流程仿真真实业务。 |
| 攻击过于危险 | 影响环境安全。 | 所有副作用进入 sink 或 sandbox。 |

## 非目标

P0 不做：

1. 生产级 IAM/SSO。
2. 生产级审批后台。
3. 真实企业 API 连接。
4. 真实公网外发。
5. 真实 HSM/TSA。
6. 完整 CI/CD 平台。
7. 完整浏览器前端。
8. 大模型自动红队。
9. 等保测评或任何形式的合规认证——文档中引用的法规和政策背景仅用于企业叙事动机，不构成靶场对合规达标的证明。

这些可以作为 P1/P2 扩展，但不能在 P0 文档或报告中写成已完成。

## 覆盖盲区

以下风险真实存在于企业中，但结构性地无法通过本靶场的 SUT adapter 观测和评测，必须如实声明，不能被暗示为"已覆盖"：

- **Shadow AI**：员工使用个人账号登录的外部大模型处理内部工作内容，数据流出企业可控边界。这类行为完全绕开 Agent Gateway 和本靶场的工具面，任何 SUT（包括 XA-Guard）都无法在这条路径上做出决策，因此也无法被本靶场评测。缓解手段（网络出口管控、终端 DLP、员工意识培训）不在 P0 范围内。
- 靶场报告中不得出现"Shadow AI 风险已由本靶场验证"或类似表述；只能表述为"已在企业场景中如实记录该风险类别及其不可测边界"。

## 结果表达边界

允许表达：

- “靶场设计已完成”。
- “P0 case 矩阵已定义”。
- “后续可接入 XA-Guard 作为 SUT”。
- “靶场与 XA-Guard 源码解耦”。

禁止表达：

- “靶场已完成运行时实现”。
- “XA-Guard 已通过靶场评测”。
- “真实企业生产环境已验证”。
- “第三方 TSA/HSM 已完成”。
- “供应链真实 marketplace 已接入”。
- “已通过等保/合规认证”。
